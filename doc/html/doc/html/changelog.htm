JmsStream Change Log

Version 2.7.9

  • Fixed minor issue with the stats GUI configuration.
  • Updated the TIBCO EMS libraries to EMS 8.2
  • Compiled with JDK 1.7_67

Version 2.7.8.1

  • No functional change. Compiled with JDK 1.6_45

Version 2.7.8

  • Fixed race condition when writing to file during async req/rep.

Version 2.7.7

  • Updated TIBCO EMS libraries to version 7.0
  • Added separate SSL parameters for JNDI and JMS connections.
  • Fixed erroneous log4j SSL messages.

Version 2.7.6

  • Removed irritating SSL info logging.
  • Updated the slf4j libraries.

Version 2.7.5

  • Fixed issue with custom properties writing to a CSV file.
  • Updated the TIBCO EMS libraries to EMS 6.3

Version 2.7.4

  • Fixed issue with publishing from a ZIP file.

Version 2.7.3

  • Fixed save file read issue with unprintable UTF-8 characters in the message body.

Version 2.7.2

  • Fixed an issue with "Stop After" for queue browsers.

Version 2.7.1

  • Fixed save file read issue if there were both a carriage return and line feed embedded in the message body.

Version 2.7.0

  • Added the ability to extract the original client message from a TIBCO EMS system monitor message.
  • Fixed an issue with leaving the connection factory blank.
  • Fixed an issue with show statistics. Changed timings from msec to sec.
  • Updated the TIBCO EMS libraries to EMS 6.1
  • Updated the Apache ActiveMQ libraries to ActiveMQ 5.5.0
  • Updated the JBoss HornetQ libraries to HornetQ 2.2.5
  • Upgraded XA support to JBoss Transaction Service 4.11.0

Version 2.6.3

  • Fixed issue saving the request response.

Version 2.6.2

  • Fixed issue using temp queues with request-response.
  • Updated ActiveMQ libraries.
  • Updated JBoss HornetQ libraries.

Version 2.6.1

  • Updated EMS libraries to EMS 6.0.

Version 2.6.0

  • Added built-in support for Apache ActiveMQ 5.
  • Added built-in support for JBoss HornetQ 2.
  • Added Apache ActiveMQ and Apache Common Logging libraries in the JmsStream distribution.
  • Added JBoss HornetQ libraries in the JmsStream distribution.
  • Fixed a bug that prevented publishing of compressed messages.
  • Fixed an issue where the -showconfig would display the configuration but continued to execute. It now stops after displaying the configuration as expected.
  • Fixed an issue with -getbodylength. The bug was introduced in 2.5.0, it now gives the correct length.

Version 2.5.2

  • Fixed issue preventing topic messages. This bug was introduced in version 2.5.1. It is now fixed.

Version 2.5.1

  • Fixed a bug preventing the sending of UTF-8 and UTF-16 character formats.

Version 2.5.0

  • Changed the embedded EMS client libraries from version 5.0 to version 5.1.4
  • Upgraded to JBoss Transaction Manager 4.9.0 GA libraries. (You must copy the libraries yourself. See docs.)
  • Changed the default save file extension to .jmss
  • Compiled with JDK 1.6.0_18 but still compatible with JRE 1.5.
  • Added the ability to read the JmsStream parameters from a Java configuration file instead of putting them on the command line.
  • Added the ability to override the sending and/or listening destinations for request/reply messages.
  • Added ability to change the default JMS client libraries to a different JMS Server.
  • Added ability for SSL connections to third-party JMS Servers.
  • Added ability to add custom JNDI settings.
  • Added SLF4j classes to distribution.
  • Fixed a bug in the Topic request/reply where the reply could only be a queue. It is now a Topic as expected.
  • Fixed a issue with writing the origination times in the original request message for request/replies.
  • Fixed a bug that prevented publishing for ZIP save files.
  • Fixed a bug that displayed erroneous "Stop..." messages for request/replies.

Version 2.4.2

  • Corrected spelling errors in the documentation and JmsStream code.

Version 2.4.1

  • Add a feature to the listener that will exit the application after receiving x number of messages.
  • Removed JBoss Transaction Manager 4.3.2 SP7 libraries due to possible legal issues. (You must now copy the libraries yourself. See docs.)
  • Removed embedded JARs from inside the EXE. Now the JARs are in the "./lib" directory.

Version 2.4.0

  • Added support for local XA Transactions through JBossTS transaction manager.
  • Added a simulated XA Transactions without using a transaction manager for performance testing.
  • Changed the embedded EMS client libraries from version 4.4.3 to version 5.0.
  • Includes the JBoss Transaction Manager 4.3.2 SP7 libraries.
  • No longer embed the third party JAR file inside the JmsStream JARs. They are now located in subdirectory and linked by the JAR manifest.
  • JmsStream.exe has all necessary JARs embedded inside the EXE.
  • Added licenses for the JBossTS transaction manager to comply with its GNU Lesser General Public License v.2.1.

Version 2.3.1

  • No new features.
  • Rearranged some code for more efficiency and added more comments.
  • Changed the embedded EMS client libraries from version 4.3 to version 4.4.3.

Version 2.3.0

  • Added JMS transaction support.
  • Added argument to use EMS message compression.
  • Compiled using JDK 1.5. JmsStream is no longer compatible with JVM 1.4.
  • Changed save file schema to conform to standard XSD and added the confirmTrans element.

Version 2.2.1

  • Added ability to capture and publish TIBCO EMS extended MapMessages.
  • Fixed application not setting message encoding.
  • Fixed application incorrectly setting all boolean message properties to false.

Version 2.2.0

  • Added -stats output for publishing messages
  • Made many internal charges to allow the JmsStream application to work with JmsStream GUI.
  • Created Windows EXE version of JmsStream.exe and JmsStreamGUI.exe

Version 2.1.1

  • Fixed null pointer error when -showconfig is the only argument.
  • Added -ssl_auth_only to force SSL encryption only for authentication.

Version 2.1.0

  • Fixed ZIP file operations. Can now read/write JmsStream capture file from/to a ZIP file.
  • Added -zipmsgperentry <number> for large message captures. It breaks up a certain number (<number>) of messages into separate ZIP entries. Good for very large message captures.
  • Changed -send <file-name> to -send <destination> to allow overriding of the message destination. Must now use -file <file-name> argument when sending messages.
  • Changed -requestreply <file-name> to -requestreply <destination> to allow overriding of the message destination. Must now use -file <file-name> argument when using request/reply messages.

Version 2.0.1

  • Fixed null pointer error when sending messages.
  • Fixed missing XML classes in JAR file.
  • Added -getbodylength arg to determine the body length when the user changes the message body.
  • Added -sequence <property> arg to include the message sequence number in a message property
  • Corrected spelling errors.
  • Slight License change.
  • Enhanced documentation.

Version 2.0.0

  • Fixed bug that caused the binary message capture to have the incorrect length.
  • Fixed bug that prevented anonymous login.
  • Fixed bug in the publishing of integer message properties.
  • Added ability to use SSL for JNDI lookups.
  • Added ability to connect to generic message factories.
  • Added ability to have fractional message rates. i.e. -rate 0.5
  • Added ability to publish messages at linearly increasing rate intervals.
  • Added ability to put message publication rate in message property.
  • Added ability to subscribe using AUTO_ACKNOWLEDGE, CLIENT_ACKNOWLEDGE, DUPS_OK_ACKNOWLEDGE, EXPLICIT_CLIENT_ACKNOWLEDGE, and NO_ACKNOWLEDGE.
  • Added ability to override message deliver mode.
  • Added usage license.
  • When publishing messages, JmsStream now loads the entire message file into memory to increase performance.
  • Removed ZIP file support because of bugs. Will reintroduce it later if needed.
  • Internal performance improvements.
  • JBoss JMS Support.
  • Revised command line options.
  • JmsStream.jar now includes the TIBCO EMS client libraries.

Version 1.2.2

  • Fixed bug in MapMessage publication.
  • Revised command line options.

Version 1.2.0

  • Added saving and replying from a text file instead of the XML format.
  • This increased publishing performance and allows large Text Type message to be replayed.
  • Fixed request/reply bugs.

Version 1.1.2

  • Fixed file write problem.

Version 1.1.1

  • Fixed program termination for publications.
  • Added receiveTime to XML File.

Version 1.1.0

  • Added request/reply.
  • Added CSV output for message headers.
  • Updated Readme.txt file.
  • Added examples.
  • Add timestamp properties for performance testing.
  • Must now use Java 1.4 or higher.
  • Added the EMS Jar files to JmsStream_App.zip for people who don't want to install EMS on their computers.

Version 1.0.3

  • Fixed agrs bug. JmsStream will now except arguments.

Version 1.0.2

  • Fixed custom connection factory and SSL bug

Version 1.0.1

  • Improved features and bug fixed to JmsStream version 0.3.2.
