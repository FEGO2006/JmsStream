JmsStream Roadmap

Planed features in future releases of JmsStream:

  • Add request/reply server ability to JmsStream.
  • Replace current JmsStream save file format with a JSON format.
  • Allow recording from multiple distinct (not wildcarded) destinations per instance.
  • Add the ability to edit TIBCO hieratical JMS MapMessage in the message editor.
  • Add feature to allow transactional bridging between two different JMS servers.
  • Add the ability to store messages in a JDBC and JDBC XA compliant Database.
  • Add the ability to sent Object Messages.
  • Add graphical monitor to JmsStreamGUI.
